    Abstract

    Here contains the notes of Frontend, including HTML, CSS, and JavaScript. The courses I took can be found in Acknowledgements.

    Keywords: HTML, CSS, JavaScript

    HTML

    HTML: the code used to structure a web page and its content, consists of a series of elements.
     Opening tag: consists of the name of the element wrapped in opening and closing angle brackets.
     Closing tag: include a forward slash before the element name.
    Attribute: contain extra information implicitly, "class" is the attribute name and "editor-note" is the attribute value. The class attribute gives a non-unique identifier to the element. The syntax for attribute setting includes:
    • a space between the attribute and the element name or between attributes.
    • the attribute name is followed by an equal sign.
    • the attribute value is wrapped by a pair of quotation marks.
    Nesting: having an effect on whatever the identifiers enclose.
     <strong> content </strong> for bold,
     <em> content </em> for italic.
    Void elements: elements containing no content.
    <!DOCTYPE html> : HTML documents must start with a type declaration
    HTML document is contained between <html> and </html>.
    The visible part of the HTML document is between <body> and </body>.
     <head> acts as a container for whatever appears after certain operations.
     <meta charset=”utf-8”>: set the character set of the document.
     <meta name=”viewport” content=”width=device-width”>: ensures the page renders at the width of viewport.
     <title>: sets the title of the page.
    <img src=”arg1” alt=”arg2” />: arg1 is the path to image file and arg2 specifies the descriptive text (blind people, screen reader)
     width=””, height=”” specifies the width and the height of the inserted picture.
    Headings: HTML headings are defined with the <h1> to <h6> tags.
    Comment: enclosed by <!-- comment -->.
    HTML paragraphs are defined with the <p> tag.
    <a href=”arg1” target=”arg2”> arg3 </a>: arg1 specifies the link, arg2 defines the way to open up a new tab, _blank for a new tab, and arg3 is the link pseudo name, a for anchor. For local links, set href to the name of the local file.
     If href=”#”, by clicking the hyperlink, it goes back to the top of the website.

    HTML tables are defined with <Table>, row as <tr> and rows are divided into data as <td>.
    HTML list starts with <ul> (unordered) and <ol> (ordered). Each item on the list starts with <li>.

    Element grouping: place the structure inside a pre-defined structure <nav>, <header>, <article>, <footer>, <aside>, <button>.
    HTML entity: predefined characters.
    Sematic HTML: HTML elements have certain meaning.

    CSS

    CSS: a language applying styles to HTML documents
     Inline CSS: writing the CSS inside the element, style=””
     Internal CSS: declare a style element <style> … </style>
     External CSS: create a .css file, use <link href=”_.css” rel=”stylesheet” />
    Selector: defines the elements to be styled
     multiple selectors can be selected at the same time, separated by commas
      selector1, selector2, … { }
     descendent selector: assign the inside selector certain properties (encoding HTML structure into CSS)
      selector1 selector2, … { }
     hash selector: use the id attribute in HTML (unique)
      id=”id_name”  #id_name { }
     class attribute: easier for multiple selection (not unique)
      class=”class_name”  .class_name { }

    Declaration: specifies the properties to be styled
    Syntax:
    • each ruleset must be wrapped in curly braces
    • the colon separates the property from its value(s)
    • the semicolon separates the declaration commands
    Box Model:
    • padding: the space around the content
    • border: the solid line outside the padding
    • margin: the space around the outside of the border
    “::”: define tag along with attribute
    Properties: color, font-size, font-family, text-transform (uppercase), font-style (italic), line-height, text-align (center) list-style (none), border: width style color (_px solid/dash color), background-color, border-top/bottom, font-weight (bold), text-decoration (none, wave), line-height
    Color:
     rgb(r, g, b): not often used
     rgba(r, g, b, transparency)
     #rrggbb
    Pseudo Class:
     selector: first/last-child { }
     select the first/last elements of the selector
     selector: nth-child(n/odd) { }
     select the nth child of the selector
     selector: link { }
     select the hyperlinks within the selector
     selector: visited { }
     select the visited hyperlinks
     selector: hover { }
     select the element being hovered by the mouse
     selector: active { }
     select the element that being interacted with the mouse
    Conflicting selectors:  declarations marked with !important > inline style > ID selector (#) > class (.)/pseudo-class selector (:) > element selector (p, div, li) > universal selector (*)
    Inherit:
     the child elements inherit the properties from the parent element
     not all properties get inherited

    Box model

     Content: text, images
     Border: a line around the element, still inside the element
     Padding: invisible space around the content, inside the element
     Margin: the space outside of the element
     Fill area: the area that gets filled with background color/image
     element width = border*2 + padding*2 + width

    Specification: padding/margin-left/right/top/bottom
     besides the actual size with pixels, one can specify the size with a ratio to the website size
    Block-level Elements: elements are formatted visually as blocks, occupy 100% of the parent elements' width
     stacked vertically by default
     body, main, header, footer, section, nav, aside, div, h1-h6, p, ul, ol, li
     display: block; converts the display method in CSS
    inline element: occupies only the necessary space for its content, causing no line breaks around the element
     heights and widths do not apply, paddings and margins only horizontally apply
     a, img, strong, em, button
     display: inline; converts the display method in CSS
    inline-block element: looks inline from the outside, behaves like block-level on the inside (images)
     occupy only content’s space, cause no line breaks, box-model apply
     display: inline-block;

    Center the website:
     1. create an element around the body of the website <container> </container>
     2. specify the width of the element
     3. set as follows:
      margin-left: auto;
      margin-right: auto;
    Positioning:
     Normal Flow: default positioning, the elements are laid out according to their order in HTML code
     position: relative;
     Absolute Positioning: use top/bottom/left/right to offset the element, with no impact on surrounding elements
     position: absolute;
     based on its location in the HTML file, its position is determined by its container (set the container relative)

    Pseudo Elements:
     indicated by double colons (::), by default an inline element
     Sibling Element: an element of the same parent
      Adjacent Sibling: the very next sibling element
      selector1 + selector2 selects the adjacent siblings of selector1 in the type of selector2
     first-letter: the first letter of the container
     first-line: the first line of the container
     after: use content to specify the content, add the content to the end of the parent element
     before: use content to specify the content, add the content to the beginning of the parent element

    Layout

     Float: make the text and inline elements wrap around the floated element (replaced by flexbox)
     float: left/right; let the surrounding elements float around this element
     the container will not adjust its height to the element
     when the child elements are all set to float, then the parent element would be funny
      solution1: add an empty child element <div></div> and set its property clear: both/left/right
      solution2: assume the parent element has class parent,
       .parent::after{
       clear: both;
       content: '';
       display: block;}
     box-sizing: border-box; merges the verge of the content box with the border

     Flexbox: applicable for any element with multiple child elements, a set of related CSS properties for 1D layouts
      empty space inside a container element can be automatically divided by its child elements (flex items)
     Flex Container: the parent element, denoted by setting display: flex;
     gap: length; create space between items, without using margin
     justify-content: align the elements vertically (flex-start/end, center, …)
     align-items: align the elements horizontally (flex-start/end, center, baseline)
     Main Axis: the horizontal direction, from left to right
     Cross Axis: the vertical direction, from up to down
     Flex Items: align-self overwrite align-items
      order: num; smaller num corresponding more to the left in order
      flex-grow: 0/1/…; automatically occupy the space within the container (the number represents multiplier)
      flex-shrink: 1/0; automatically change the width of flex items so that they fit in the container (1 for yes)
      flex-basis: num; set the width of flex items to num
      flex (grow) (shrink) (basis) combines the upper 3 commands

     CSS Grid: denoted by display: grid; the elements are displayed in columns and rows

     Grid Container:
      grid-template-rows: size; establish the grid row tracks
      grid-template-columns: size; establish the grid column tracks
      row-gap: length; create empty space between row tracks
      column-gap: length; create empty space between column tracks
      justify-items: stretch/…; align items inside rows
      align-items: stretch/…; align items inside columns
      justify-content: start/…; align the grid inside the grid container horizontally (container > grid)
      align-content: start/…; align the grid inside the grid container vertically (container > grid)
     Grid Items:
      grid-column: start_line/end_line; place a grid item into a specific cell
      /span _ specify the length that a cell span
      /-1 all the way to the end (double labelling)
      grid-row: start_line/end_line;
      justify-self overwrite justify-items
      align-self overwrite align-items
     For flexible size, use _fr , which stands for fraction, representing the relative size
     repeat(_times, content); repeat content for designated times

    Web Design

    Typography

    Typeface: limit to utmost 2 in one webpage
     Serif: typefaces that contain decorative lines, in comparison with sans-serif
     Typeface resources:
      google fonts
      1001 fonts
    Font Size: limit between 16px and 32px
     for long text, try a size of 20px or bigger
     headlines can be set to 50+ px and 600+ bold
     for any text, try to avoid a font weight under 400
     Resource:
      type to scale
    Reading Friendly:
     less than 75 characters per line (65-72)
     for normal-sized text, use a line height between 1.5 and 2, for big text, below 1.5
     The smaller/longer the text, the larger the line height
     if the headlines look unnatural, try to decrease letter spacing
     use all caps for short titles, make them small and bold, and increase letter-spacing
     usually, the text shouldn’t be justified
     don’t center long text blocks
     use the <link> element to import the style from the source first
     parameters: font-size, line-height, letter-spacing

    Color

    Color System:
     a main color and a grey color is recommended
     accent color should be determined by the main color and the grey color
     for diversity, create lighter and darker versions (tints and shades) for each color
     main color should be used to draw attention to the most important elements on the page
     on dark-colored backgrounds, try using a tint of the background for text
     text should usually not be completely black (larger ones)
     never make the text too light, check the contrast between the text and background colors
     contrast ratio: 4.5 : 1 for normal text and 3 : 1 for large text (18px+) at least
     Resources:
      open-source color scheme
      color palettes generator
      Tailwind CSS
      paletton
      flat ui color palatte

    Images and illustrations

    Types: product photos, storytelling photos, illustrations, patterns
    Text on images:
     darker/brighten the image
     position text into a neutral image area
     put text in a box
    to account for high-res screens, make image dimensions 2x as big as their displayed size (physical pixel)
    compress images for lower file size and better performance
    when using multiple images side-by-side, make sure they have exactly the same dimensions
    Resources
     hippopx
     pinterest
     behance
     dribble
     pexels
     pixabay
     unsplash
     undraw

    Icons

    Use SVG icons or icon fonts but not bitmap image formats (jpg, png), copy the svg code
    Do not make icons larger than what they were designed for
     stroke: color; set the color of the svg icon
     width/height: len; set the width and height of the icon
     fill: color; fill the svg icon with the specified color
    Resources
     heroicons
     phosphoricons
     ionicons
     icons8

    Shadows

    Use shadows in small doses
    Go light on shadows
    Small shadows for smaller elements that should stand out
    Medium-sized shadows for larger areas that should stand out more
    Large shadows for elements that float above the interface
    box-shadow: horizontal_offset vertical_offset blur (scale) color;
     color should be specified using rgba()
     For text, use text-shadow

    Border Radius

    border-radius: len; specify the radius of the border
    border-bottom/top-left/right-radius: len; specify the radius of one angle

    Whitespace

    Start with a lot of whitespace and remove them
    Try a hard rule, using multiples of a certain quantity for all spacing

    Visual Hierarchy

    Position important elements closer to the top of the page

    Responsive Design

    1. Fluid Layout: allow webpage adapt to the current viewport width
     use %/vh/vw unit
     use max-width instead of width
    2. Responsive Unit: use rem unit instead of px for most lengths to scale the layout automatically
     setting 1rem to 10px for easy calculation
    3. Flexible Image: images by default don’t automatically adapt to the viewport
     use % for image dimensions, together with the max-length property
    4. Media Queries: change CSS styles on certain viewport widths (breakpoints)

    Web Introduction

    Router: a device that allocates the flow of information
    Modem: convert the information from the network into information manageable by the telephone infrastructure and vice versa
    ISP: a company that manages special routers that are linked together and can access other ISPs’ routers
    Intranets: private networks that are restricted to members of a particular organization
    Client/Server: computers connected to the internet
     Client: typical web users’ internet-connected devices
     Server: computers that store webpages, sites, or apps
    TCP/IP: communication protocols that define how data should travel across the internet
    DNS: a dictionary of IP addresses with domain names
    HTTP: an application protocol that defines the communication between clients and servers specifies how the hypertext (linked web documents) are transferred
    Component file parse order:
     Browser parses HTML file and recognizes the references
     Browser sends requests for CSS files and JS files in <link> and <script></script>
     browser generates an in-memory DOM tree, CSSOM structure, compiles and executes the JS
    Web Server: a computer that stores web server software and a website’s component files
     Includes several parts that control how web users access hosted files
     Static: consists of a computer with an HTTP server
     Dynamic: consists of a static web server plus extra software (application server and a database). Updates the hosted files before sending content to the browser
    URL: a text string that specifies where a resource can be found on the internet
    FTP: an insecure protocol for transferring files from one host to another over the internet
    Resources
     validator checks the validity of the website
     diffchecker checks the difference between texts

    JavaScript


    Releases: ES6: biggest update, from then on, the language is called modern JavaScript with new updates every year
    Linking:
     Inline Script: <script> in the HTML file contains the JS contents
     External Script: a separate file with .js as extension, <script src=”path”> links the file

    Declaring Variable:

     Variables declared by let are only available inside the block where they're defined
     Variables declared by var are available throughout the function in which they're declared
     let var1 = value1; assign value1 to the variable var1
     var1 = value2; reassign var1 with value2
     const var1 = value2; declare a constant var1 with value2 as value
     var var1 = value1; declare a variable var1 with value1 as value
     console.log(value1/expr1, …); output the value to the weblog
     The variable name must abide by certain syntax
      can only contain letters, numbers, underscore, and the $ sign
      not the same as reserved keywords

    Data Types:

    The values in JS are either an object or a primitive value
    Dynamic Typing: data types are determined automatically by JS (note that the values have type, rather than variables)
    Primitive:
     1. Number: only floating-point numbers
     2. String: identified by single/double quotes
     3. Boolean: true and false
     4. Undefined: a variable defined but not assigned with value (let var1;)
     5. Null: means empty value
     6. Symbol: value that is unique (since ES2015)
     7. BigInt: larger integers than the number type can hold (since ES2020)
    typeof value returns the type of the value
    Type Coersion:
     let n = '1' + 1; converts the latter 1 into a string, the plus sign is interpreted as the concatenation operator

    Operators:

    Operator Precedence
    Template String:
     Denoted by backticks (``), able to concatenate strings using the syntax
     con_str = `str1 ${var1}`, the placeholder ${expr1} would be replaced by the value of expr1
    Decision Making: if (condition) { command1 } else if { command2 } else { command3 }
    Falsy Values: values that would become false when being converted: 0, '', undefined, null, NaN
     Other values are called the truthy values
    Equality Operator: === strict, true only if the 2 values are identical, no type coercion
     In parallel, the strict inequality operator is !==
    Logical Operator: and &&, or ||, not !var1
    Conditional Operator: condition ? command1: command2;
     If the condition before the question mark returns true, then command1 would be executed, otherwise command2
    Strict Mode:
     'use strict'; in the first line activates the strict mode

    Expression and Statement:

    Expression: pieces of code that produce value
    Statement: pieces of code that do not produce value
    Switch Statement:
     switch (var1) {
      case ‘case1’: command1;
       break;
      case ‘case2’: command2;
      default: command3;}
     Without the break statement, the code carries on

    Function:
     Function Declaration: can be called before the function is defined
      function func_name(parameters) {commands; returns;}
     Function Expression: can only be called after the function is defined
      const func_name = function(parameters) {commands; returns;}
     Arrow Function: a special form of function, the return is implicitly included
      const func_name = (parameters) => {commands; returns;}

    Array: denoted by square brackets [ ], the elements are separated by commas
     Can be defined using array_name = new Array(param1, …);
     Indexing is also with square brackets, array_name[index];
     .length returns the number of elements contained in the array
     .push(param) add the parameter param to the end of the array, returns the new length of the array
     .unshift(param) add the parameter param to the beginning of the array
     .pop() remove the last element from the array, returns the popped element
     .shift() remove the first element from the array, returns the popped element
     .indexOf(param) return the index of the param that is inside the array
     .includes(param) return true if param is inside the array, otherwise false

    Objects:
     Indicated by curly braces, defining key-value pairs, each key is also called a property
     obj = {key: pair, …}
     indexing: obj.key or obj[‘key’ returns the pair corresponding to key
     updating: obj.key = pair1;
     appending: obj[‘key’] = pair;
    Method: any function attached to an object
     func_name: function (params) { commands; returns;}
     Inside the object, keyword this can be used to refer to the object itself

    Loop:
     for (let rep = init_value; rep cond expr; rep++) {command;}
     continue and break are equal as they are in C
     while (condition) {commands;}

    DOM:
     Documented object model, structured representation of HTML documents, allowing being accessed by JS
     document.querySelector(‘identifier’) select the element named identifier, the same as is located in CSS
     .textContent get the text of the selected element
