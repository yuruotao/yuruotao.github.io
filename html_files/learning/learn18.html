    Abstract

    An embedded system is a microprocessor-based computer system designed to perform a specific function within a larger mechanical or electrical system. The most common types of embedded systems include consumer electronics, medical devices, automobiles, and industrial process control.

    This article is the note for embedded system learning.

    Keywords: Embedded System

    嵌入式系统

    嵌入式系统概述

    嵌入式系统：为特定应用开发的专用计算机系统
    • 特点：专用性、稳定性、资源受限、高可靠性、实时性、软件固化；
    • 分类：
      • 低端：4/8位，无操作系统，监控程序控制；
      • 中端：8/16位，小型操作系统，UI简单；
      • 高端：32/64位；
    • 发展：
      • 20世纪60年代初：阿波罗导航计算机，第一个公认现代嵌入式系统；
      • 20世纪60年代中：嵌入式系统批量生产；
      • 20世纪70年代：微处理器出现后真正意义上的嵌入式系统发展；
    存储器：
    • 易失性存储器：SRAM，DRAM
      FRAM：铁电RAM，MRAM：磁性RAM
    • 非易失性存储器：
      • 电可擦可编程只读EEPROM
      • 闪存Flash ROM
      • 磁盘，光盘存储器
    嵌入式CPU：
    • 特点：实时处理、低功耗、结构可扩展、集成测试电路
    • 类型：
      • 微处理器：PowerPC，MC68000，MIPS，AMD，x86
      • 数字信号处理器（DSP）：处理数据信号的微处理器芯片，支持单指令多数据并行处理的指令
      • 微控制器（MCU）：集成在单个芯片的小型计算机，包含CPU与内存
        数码相机中一般使用8位MCU进行操作控制
      • 片上系统（SoC）：集成电路加工工艺产物，高端嵌入式系统，可集成数字、模拟电路
    微电子技术：
    • 以半导体技术为核心
    • 分类：
      • 大规模（LSI）：3k-10万
      • 超大规模（VLSI）：10万-100万
      • 极大规模（ULSI）：大于100万
      • 嵌入式芯片一般属于VLSI与ULSI
    集成电路：
    • 特点：体积小、重量轻、可靠性高
    • 工作速度主要取决于组成逻辑门电路晶体管尺寸（极限工作频率）
    • 芯片上电路元件线条越细，相同面积晶片可容纳更多晶体管
    SoC芯片：
    • 分类：
      • 通用：芯片中可包含一个或多个CPU
        自主开发处理器内核或购买其他企业处理器内核的IP授权
      • 专用：半导体企业定制的芯片
    • 开发流程：
      1. 总体设计：RTL语言描述
      2. 逻辑设计：逻辑综合软件将源文件进行综合
      3. 综合与仿真
      4. 芯片制造
    IP核：
    • 知识产权属于某相关团体的可复用设计单元（library）
    • 核：完成并经过验证的集成电路
    • 复用：所有SoC芯片包含CPU（以IP核形式集成于芯片）
    • 级别：
      • 逻辑门级：基本门电路
      • 寄存器传输级：寄存器、多路选择器、译码器
      • 行为/功能级：CPU、DSP、存储器、总线与接口电路
    • 分类：
      • 软核：灵活、可移植性（HDL）
      • 固核：RTL
      • 硬核：保证性能、开发工作量少（IC设计文件）
    • CPU内核：ARM，MIPS、PowerPC
      32位嵌入式处理器主要采用英国公司设计的ARM内核
    • ARM内核授权内容：
      • 芯片内核设计技术
      • 软件开发工具
      • 含ARM CPU的芯片销售权

    计算机基础知识

    文本：
    • 基于特定字符集的一个字符流，每个字符均用二进制表示
    • 汉字：GB2312（双字节），GB18030（向下兼容）
    • Unicode：
      • UTF-8：单字节可变长编码，一般采用3字节编码
      • UTF-16：双字节可变长编码，一般采用双字节编码
      • GB18030与Unicode基本兼容，编码不一定相同
    • 分类：
      • 简单文本：由字符编码组成，几乎不包含其他格式信息和结构信息（.txt）
      • 丰富格式文本：doc，html，pdf
    • 展示：打印输出、屏幕阅读
    • 字符形状描述：点阵法、轮廓法
    图像：
    • 获取：
      1. 扫描：划分网格
      2. 分色：取样点的颜色分为三个基色（RGB）
        灰度图像不用分色
      3. 取样：测量取样点各基色的亮度值
      4. 量化：对各取样点亮度值进行AD转换
    • 参数：
      • 分辨率
      • 位平面数目：像素颜色分量的数目
      • 像素深度：每个像素用多少个二进制位表示
      • 颜色模型：RGB，CMYK
      • 数据量：分辨率*像素深度/8（单位为字节）
    • 压缩类型：
      • 无损压缩：BMP，TIF，GIF，PNG
      • 有损压缩：JPEG
    音频：
    • 声音信号数字化：取样、量化、编码
    • 参数：
      • 码率：每秒钟的数据量
      • 压缩前码率=取样频率*量化位数*声道数
    • 压缩：
      • 无损压缩：FLAC，APE，M4A
      • 有损压缩：MP3，WMA，AC3，AAC
    视频：
    • MPEG-1：VCD，数码相机（MP3为三层算法）
    • MPEG-2：用途最广，DVD，数字卫星电视转播，数字有线电视，数字视频传输
    • MPEG-4：应用最多，光盘，iPhone，XBOX
    通信：
    • 要素：信源、信道、信宿
    • 分类：有线/无线
      微波：手机、卫星、WiFi
    • 传输技术：
      • 调制：利用信源信号改变载波的某个参数
      • 解调：接收方将信号恢复为原始信号
      • 时分多路复用（TDM）：轮流使用同一线路传输数据
      • 频分多路复用：信源的信号调制在不同频率的载波上
      • 电路交换：通话全部时间内用户始终占用端到端的传输信道
      • 分组交换：提高传输线路利用率
    计算机网络：
    • 分类：
      • 局域网（LAN）：为一个单位所有，地理范围有限
        使用专门铺设的传输介质进行联网和数据通信（主要用双绞线）
        数据传输速率高，延迟时间短，误码率低
      • 城域网（MAN）
      • 广域网（WAN）
    • 以太局域网：计算机网络的一种形式，以集线器或交换机为中心
    • 分组交换技术：局域网采用
      发送设备MAC，接收设备MAC，控制信息，有效载荷（传输的数据），校验信息
    • 局域网分类：传输介质（有线，无线）、拓扑结构（星型，环形），访问控制方法（以太网、FDDI、令牌网）
    • 以太网数据帧：前导码、帧起始定界符、目的MAC、源MAC、类型/长度、数据、帧校验序列
    • 无线局域网：以太网与无线通信技术结合的产物
      特点：
      • 方便地移动计算机位置或改变网络的组成
      • 借助无线电波传输（2.4GHz，5.8GHz）
      • 协议为IEEE 802.11（WiFi），数据传输速率为11，54，108Mbps
      • 需使用无线网卡，无线接入点为热点
    互联网：
    • 结构：最初：主干网、地区网、校园网
      目前：基于互联网服务提供商（ISP）的多层次结构
    • 接入：城域网的运营商作为ISP承担用户接入任务
      城域网主干为光纤传输的高速宽带网
    • 互联：将同构、异构的网络互相连接（IP协议、路由器）
    • IP地址：所有主机使用独特的IP地址进行标识
      • IPv4：每个IP地址用4B表示，包含网络号（从属的物理网络）、主机号（物理网络中的编号）
      • 点分十进制：一个IP地址用4个十进制数表示
      • IP地址分类包含ABC三个基本类，C常用于小规模局域网，每个网络最多包含254台计算机，DE备用
        A类IP地址由1字节网络地址，3字节主机地址构成（最高位为0）
      • IPv6：扩展IP地址为128位
    • IP数据报：IP协议定义的独立于物理网的数据包的格式
      • 所有在TCP/IP网络传输的数据在IP层面均需为IP数据报格式
        IP协议：在Internet中负责选择合适路由，使packet找到目标机
      • 头部信息：源IP，目的IP，协议版本等控制信息
      • 数据区：最大（包含头部信息）64KB
    • 路由器：遵循IP协议将异构网络互联的设备，功能主要包含选择路由、转发IP数据报、协议转换
      路由器有多个I/O端口，路由器之间一般用高速通信链路连接，一个路由器可有多个IP地址
    • 接入：
      • ADSL：利用普通电话线为传输介质，有效传输距离3-5km
      • 有线电视网：利用光纤同轴电缆混合网传输电视节目
      • 光纤接入：使用传输介质的互联网接入系统
      • 无线接入：必须有可用的AP（接入点）

    嵌入式处理器

    嵌入式处理器概述

    结构类型 ：
    • 指令集：复杂指令集结构（CISC），精简指令集结构（RISC）
    • 存储机制：冯诺依曼结构，哈佛结构
    • 字长
    • 内核：51，ARM，MSP430
    指令集/IS架构：设计处理器的主要依据
    • RISC指令字长统一，指令操作简单，执行时间短
    • RISC与CISC非对立
    存储结构：
    • 哈佛结构用两个独立存储器分别存储指令和数据
    • 哈佛结构使用两条总线，分别作为CPU与存储器间专用的通信路径

    ARM处理器

    • 特点：
      • 单周期操作
      • 只使用load/store访问内存
      • 指令长度固定
      • 三地址指令格式
      • 指令流水线
    • 指令集：
      • ARM指令集：全部为32位指令，可通过协处理器扩展功能
      • Thumb指令集：全部为16位字长的指令，ARM指令的压缩形式
      • Thumb-2指令集：Thumb的超集，包含32位指令
    • 分类：经典ARM，ARM Cortex-A，ARM Cortex-R，ARM Cortex-M，ARM专家处理器
      • Cortex-A：application，面向高端应用，高计算要求，需运行功能丰富的操作系统等
        除A50为64位字长，其他均为32位
        提供Java字节码加速执行的Jazelle技术
      • Cortex-R：real time，面向实时控制，安全，低成本
      • Cortex-M：microcontroller，面向微控制器，只支持Thumb-2的子集
        CM0，CM0+：成本低（体积最小，能耗最低），简单
        CM3：工业控制与中高端应用
        CM4：有效的数字信号控制功能
      • 经典：ARM7：冯诺依曼，32位，诺基亚6110
        ARM9：数字信号处理，ARM920T（ARMv4T架构）

    ARM处理器工作状态

    • 处理器工作状态：
      • ARM状态：32位状态，指令均为32位宽度
      • Thumb状态：16位状态
        开关外部中断：CPSIE I/CPSID I
      • 调试状态
      除Cortex-M系列外，ARM处理器复位后开始执行代码时总是处于ARM状态
      Cortex-M处理器只有Thumb-2指令集，无ARM状态
    • ARM与Thumb切换：
      • ARM-Thumb：BX指令将操作数寄存器最低位设置为1
      • Thumb-ARM：BX指令将操作数寄存器最低位设置为0
    • ARM工作模式：全部工作模式见表2
      • CPSR：程序状态寄存器，低五位决定工作模式
      • 工作模式转换条件：处理器处于User模式时，除非发生异常，不能改变工作模式

    ARM状态下的寄存器组织

    • ARM处理器共有37个寄存器，31个通用，6个状态（CPSR，SPSR），均为32位
    • 工作于ARM状态下，物理分配上寄存器被安排为部分重叠的组，每种模式使用寄存器不同
    • 共同点：
      • R15均作为PC使用（程序计数器）
      • CPSR为当前程序状态寄存器
      • R7-R0为公用的通用寄存器
    • 通用寄存器中不分组的寄存器有8个：R0-R7
    • R8-R12共两组，10个寄存器，fiq代表快速中断模式专用，与其他模式地址重叠
    • R13，R14在User与SYS下为堆栈指针，程序链接寄存器，其他模式下有独特的标记方式
    • SPSR为保存处理状态寄存器，在当前工作模式下保存处理状态

    程序状态寄存器（CPSR）

    • N：符号标志，N=1为负数
    • Z：全零标志，Z=1为全零
    • C：进借位标志，C=1加法进位/减法无借位
    • V：溢出标志，V=1为有溢出
    • Q：增强的DSP运算指令是否溢出标志，Q=1溢出
    • I：中断禁止控制位，I=0允许IRQ中断
    • F：禁止快速中断控制位，F=0允许FIQ中断
    • T：ARM与Thumb指令切换，T=1执行Thumb，否则执行ARM
      对不具备Thumb指令的处理器，T=1表示强制下一条执行的指令产生未定义的指令中断
    • M：模式选择位

    异常/中断

    进入复位状态时，系统将PC指向0x00000000处，执行系统的第一条指令
    • 异常响应：
      1. CPSR保存至要执行的异常中断对应的各自SPSR中
      2. 设置CPSR相应位：M4-M0对应工作模式，I=1禁止IRQ中断，若进入复位/FIQ，设置F=1
      3. 将引起异常指令的下一条地址保存至新的异常工作模式的R14中
      4. 给PC赋值，转入异常向量地址执行
    • 异常返回：复位异常处理程序执行完无需返回
      1. 恢复用户寄存器，SPSR_mode寄存器值复制到CPSR
      2. PC值恢复为断点地址
      3. 清除CPSR的I，F，开放IRQ，FIQ

    存储器

    • 32位ARM处理器支持字节、半字、字三种数据类型，字、半字需对齐（地址低2位为0）
    • 存储字格式：
      • 大端模式：32位数据字高字节存储在低地址，低字节存放在高地址
      • 小端模式：高字节存储在高地址
    • 虚拟存储器：借助MMU，将系统中不同类型的存储器进行统一管理，通过地址映射，使需要运行在连续地 址空间的软件可运行在不连续的物理存储器中的技术
    • MMU：存储器管理单元
      • 功能：虚拟地址到物理地址映射（页表，虚拟地址与物理地址的对应表）
        存储器访问权限控制
      • TLB：页表的高速缓存
      • 存储器访问顺序：
        MMU先查找TLB中的转换表，若无，查找主存储器内的页表，若无，产生中断，OS进行处理
      • MPU：存储器保护单元，可替代MMU管理存储器（段式）

    ARM指令：（每条指令均为32位）

    • 格式：OPCODE{CONDITION}{S} Rd, Rn {,OPR2}
      • {}中内容可省去，opcode、cond，S间无分隔符，{S}与Rd间用空格隔开
      • OPCODE：指令的操作码/注记符（条件码）
      • {CONDITION}：条件域
      • {S}：指令执行时是否更新CPSR，有S时更新
      • Rd：目的寄存器（任意通用寄存器）
      • Rn：第一个源操作数（限制为任意通用寄存器）
      • OPR2：第二个源操作数（寄存器、任意移位寄存器）
    • 程序的启动从ARM指令集开始，进入异常转化为ARM状态
    • 条件域：每条ARM指令包含4位条件码，位于指令的最高4位，共16种
    • 操作数符号：
      • #：立即数符号
      • 0x：十六进制符号
      • ！：更新基址寄存器符号
      • ^：复制SPSR到CPSR符号
      • -：寄存器列表范围符号，从…到…
    • 寻址方式：
      1. 立即寻址：操作数本身在指令中给出（立即数以#为前缀）
      2. 寄存器寻址：寄存器中的数值为操作数
      3. 寄存器间接寻址：操作数存放
        在存储器中，寄存器用[ ]作间接寻址
      4. 基址加变址寻址：将基址寄存器中的内容与指令中给出的地址偏移量相加得到有效地址
      5. 相对寻址：以PC当前值为基址，指令中地址标号为偏移量，相加得到有效地址
      6. 堆栈寻址：递增堆栈：堆栈由低地址向高地址生成
      7. 块拷贝寻址：多寄存器寻址，一条指令传送最多16个通用寄存器的值

    ARM指令集

    • 数据处理类：
      • MOV{cond}{S} Rd, OPR2 ;数据传送，将OPR2值赋给Rd
      • MVN{cond}{S} Rd, OPR2 ;取反传送，将OPR2取反后的值赋给Rd
      • CMP{cond}{S} Rd, OPR2 ;数据比较，Rd-OPR2，改变CPSR中标志位
      • CMN{cond}{S} Rd, OPR2 ;反值比较，Rd+OPR2，改变CPSR中标志位
      • TST{cond}{S} Rd, OPR2 ;位测试，Rd AND OPR2，改变CPSR中标志位
      • TEQ{cond}{S} Rd, OPR2 ;相等测试，Rd EOR OPR2，改变CPSR中标志位（同或）
      • ADD{cond}{S} Rd, Rn, OPR2 ;加法，将Rn+OPR2的值赋给Rd
      • ADC{cond}{S} Rd, Rn, OPR2 ;带进位加法，将Rn+OPR2+!C的值赋给Rd

    基于ARM内核嵌入式芯片硬件组成

    • 存储器：
      • 片内程序存储器常用Flash ROM，片内数据存储器常用SRAM（Cache）
      • 程序存储器大都采用Flash存储器，数据可采用SRAM，DDR，DDR2，DRAM
      • 高带宽外部存储器控制接口可扩展程序、数据存储器，与系统总线部分相连
    • 中断控制器：
      处理中断方式：
      • 标准中断控制器：发送中断请求信号给处理器核
      • 向量中断控制器：读取中断控制器与各设备对应表示中断请求的寄存器
      中断管理方式：
      • 向量中断（VIC）：新中断的优先级高于当前中断，才向内核提出中断请求
      • 嵌套向量中断（NVIC）：可进行中断嵌套
        仅Cortex-M支持嵌套的向量中断
    • DMA控制器（直接存储器访问控制器）：
      • 一种硬件组件，可将数据块从外设——内存/内存——外设/内存——内存
      • 数据传输过程不需CPU参与，降低功耗
      • ARM与通道进行交流时，可由ARM内核/DMA控制数据传输
    • 电源管理与时钟管理器：
      • 时钟控制器：负责对时钟的分配，产生不同频率定时时钟作同步时钟
      • GPIO：通用I/O端口，作为输入时有缓冲功能，作为输出时有锁存功能（引脚复用）
        n个GPIO引脚最多能构成2^n个按键的键盘
    • 定时计数组件：
      • 看门狗（WDT）：处理器进入错误状态后一定时间内复位
      • 通用定时器（Timer）：用于一般的定时
      • RTC：提供年月时分秒，使系统具有独立的日期和时间
      • PWM
    • 数字-模拟模块：
      • ADC：完成模拟信号到数字信号的变换
      • DAC：完成数字信号到模拟信号的变换
      • 比较器：对模拟电压信号等与基准信号比较（监测电压、温度等）
    • 互联通信组件：
      • 通用异步收发器（UART）：标准的串行通信接口，低位在前高位在后
      • I2C：集成电路互联的一种总线标准，2根信号线：时钟线SCL，数据线SDA，可完成同步半双工串行通信
      • I2S：针对数字音频设备间的数据传输的音频串行总线
      • SPI：使MCU与外围设备以串行方式进行通信，四线制：MISO，MOSI，SSL，SCK（板级芯片通信）
        M与S表示主机、从机
      • CAN：控制器局域网，差分传输，CANH/L数据线，要求抗干扰能力强的工业控制领域
      • USB
      • 以太网（RJ-45连接器）

    嵌入式系统存储器

    层次结构
    • 处理器内部通用寄存器速度最快（Flash ROM）
    • CPU使用最频繁的代码、数据用SRAM作Cache存放
    • 运行中的程序、数据放在主存储器
    • 其他存放于外存
    分类：
    • 随机存取存储器
      • SRAM：静态，处理器内嵌的数据存储器
      • DRAM：动态，外部扩展的存储器
      • DDRi SDRAM：2^i倍数据速率技术，预读2^i位数据
    • 只读存储器
      • PROM：单次可编程存储器，无法修改
      • EPROM：紫外线可擦除可编程存储器
      • NOR/NAND Flash：主要闪存技术
        NAND：以页/行位单位随机存取，容量、使用寿命、成本占优
        NOR：以字节为单位随机存取，速度占优，写入擦除速度慢
    性能指标：
    • 存储器容量：能够存储的二进制位数
      内存容量 = 单元数 * 数据位数 / 单元
    • 存取时间：单位一般为ns
    • 带宽：每秒可传输的最大数据总量
      并行总线带宽 = 总线频率 * 数据宽度 / 8 * 传输次数 / 总线周期
      串行总线带宽 = 总线频率 / 10
    片内存储器：嵌入式处理器内部嵌入的存储器
    • 片内Cache：减少访问外部存储器次数
    • 片内Flash ROM：使系统以最小系统形式应用
    • 片内Flash SRAM：临时数据存储，复位后需初始化
    • 片内FRAM：可当RAM，也可当ROM
    片外程序存储器
    • NOR Flash ROM
    • NAND Flash ROM
    片外数据存储器：SDRAM, DDR
    韩国DDR存储器命名：
    HYXZmnjk
    • HY：现代标识
    • X：存储器类型，5，57为SDRAM，5D为DDR
    • Z：电压等级，U为2.5V，V为3.3V，缺省为5V
    • m：
      • SDRAM：总容量，56，52为256Mb，64，65为64Mb，26，28为128Mb
      • DDR：28为128Mb，56为256Mb
    • n：数据宽度（16/32）
    • j：逻辑BANK数，2^n个BANK
    • k：电气接口（0：LVTLL，1：SSTL，2：SSTL_2）
    外部辅助存储设备：
    • USB闪存盘：无需物理驱动器，即插即用
    • 微硬盘：有缓存，无需外置电源，高防振性、高速传输，高达百GB

    I/O接口与常用I/O设备

    GPIO：通用输入输出接口
    • 所有嵌入式处理器内部集成
    • 输入缓冲、输出锁存，一般具有三态
    • 复用引脚
    I2C：集成电路互联总线
    • 用于连接嵌入式处理器与外围器件，采用串行半双工标准（任意时刻信号单向传递）
    • ARM芯片内部集成I2C总线接口
    • 主动发起数据传输操作的器件为主控器件
    • 仅有两条信号线
      • 数据线SDA：高低电平状态仅在SCL为低时可改变，检测电平与自己电平相同则继续占用总线
      • 时钟线SCL
    • 数据传送：
      • 总线无请求时，两信号线处于高电平
      • 数据传送时高位在前，低位在后，传送字节数无限制，第一字节为地址
      • 接收方应发送确认信号ACK
      • 属于多主总线，允许多个主控器件同时操作
    SPI：串行外设接口
    • 同步串行外设接口，使处理器与外围设备以串行方式进行通信（数据交换）
    • 全双工串行外设接口
    • 一般使用4条线
      • SCK：串行时钟线
      • MISO：主机输入/从机输出数据线
      • MOSI：主机输出/从机输入数据线（可能没有）
      • SSEL：低电平有效的从机选择线
    • 连接方式：
      • 一主一从式
      • 互为主从系统（主机设置SSEL为低电平）
      • 一主多从式（SPI所有信号为单向）
      • 多主多从式
    UART：串行异步通信接口通用异步收发器
    • 常用于全双工串行异步通信
    • 所有嵌入式芯片均继承兼容标准UART功能的组件
    • 组成：
      • 发送器：先到缓冲寄存器，再通过移位寄存器按位发送
      • 接收器：FIFO或普通模式接收，FIFO缓冲器满引发接收中断，普通模式一个字符引发接收中断
        一般接收缓冲器、发送缓冲器FIFO字节数相同，可编程选择长度，不同ARM芯片最大字节数不同（ARM9的S3C2440为64B）
      • 波特率发生器：最高为115200b/s
    • 通常接收采用中断，发送采用查询，直接UART通信限于版/芯片间通信，其他情况需用逻辑电平转换接口（RS-232）
    • 最高数据传输速率一般低于SPI
    • 传送信息格式以起始位开始，停止位结束，长度为5-8位
    USB：通用串行总线
    • 支持设备的即插即用、热插拔
    • 通信采用主从方式
    • USB OTG：直接在具有OTG的USB总线上完成通信，没有主机的情况下完成设备间数据传送
    • 接口提供内置电源，向低压设备提供5V电源
    • 组成：
      • 硬件：根集线器，提供USB下行端口
      • 软件：USB设备驱动程序、USB驱动程序、USB主控制器驱动程序
    • 采用翻转不归零编码方式（NRZI），电平保持为1，电平翻转为0
    • 总线采用半双工差分方式，有4根信号线：2根差分、电源线、地线
    HDMI：高清晰度多媒体接口
    • 最高数据传输速率为10.2Gb/s
    • 可同时传送音频、视频信号，无需进行A/D、D/A转换
    键盘：
    • 分类：
      • 线性键盘：一个按键占用一个引脚
      • 矩阵键盘：行列矩阵形式构成键盘，读取按键特征值决定按键结果
    • 简单键盘可由UART制作
    • 机械式键盘按下会产生抖动
    触摸屏：
    • 分类：
      • 电阻式：价格低
      • 电容式：价格高，形成耦合电容
        一般与ARM芯片采用USB/I2C总线方式连接
        支持多点触摸或滑动操作
    发光二极管：与ARM芯片连接只需一根GPIO引脚，但需加驱动电路（电流有限）
    LED数码管：共阳极连接与共阴极连接（共阳极1为灭）
    LCD显示设备：
    • LCD显示屏、LCM、LCD显示器
    • LCD显示屏不具有控制器与控制电路
    • 接口标准主要为VGA与DVI

    嵌入式系统软件

    嵌入式软件特点：
    • 具有较长的生命周期
    • 受限于功能和具体应用环境
    • 软件固化存储
    嵌入式软件结构：
    1. 简单的轮询结构：不需要操作系统支持，应用直接在裸机上运行
      • 程序结构简单，便于编程
      • 无中断机制，可靠性较高
      • 只适用于系统任务量少、任务处理简单、实时性要求低的场景
    2. 带中断的轮询结构：提高对紧急事件的相应速度，可并发处理不同的异步事件
    3. 监控式操作系统 + 应用软件结构：
      • 减少开发时间
      • 引入操作系统增加系统的开销
      • 降低开发嵌入式软件的复杂度，有效保证系统实时性、可维护性
    4. 通用嵌入式操作系统 + 应用软件结构：
      • 硬件抽象层/驱动层
      • 操作系统层
      • 中间件层：除操作系统内核、设备驱动程序、应用软件外的系统软件
    硬件抽象层与板级支持包：
    • 硬件抽象层（HAL）：操作系统与硬件间设置的独立接口软件层，是所有直接依赖于硬件的软件
      对硬件进行抽象，增强上层软件可移植性
      HAL系统库：
      • 提供标准库函数
      • 提供设备驱动，使用户可访问所有设备
      • 提供HAL API
      • 进行系统/设备初始化
    • 板级支持包（BSP）：硬件抽象层在特定操作环境下的具体实现
      • 具有硬件相关性、操作系统相关性
      • 规范为OS相关的
      • 代码需针对特定硬件专门编写
      • 与硬件平台直接相关，调试有最小系统调试、外围设备驱动程序调试
      • 功能：初始化、设备驱动、引导加载
    • 实例：
      • Nios 2 HAL：32位Harvard结构通用RISC处理器
      • Windows Embedded Compact 7：专门为小型网络设备设计的操作系统
    引导加载程序Bootloader：
    • 嵌入式系统上电后首先运行引导加载程序，负责：
      • 系统的上电自检
      • 硬件初始化
      • 建立存储空间映射
      • 配置系统参数
      • 建立上层软件运行环境
      • 加载和启动操作系统
    • Bootloader依赖具体硬件结构，一般对不同硬件结构有不同版本
    • 引导加载方式：
      • 启动加载模式：
        1. 在ROM中直接运行操作系统代码
        2. 在RAM中运行操作系统代码
        3. 外存储器加载操作系统代码运行（主流）
      • 下载模式：从通信端口加载操作系统代码运行
    • 引导加载程序执行过程：
      • 第一阶段（汇编）：
        关闭终端
        处理器内部基本REG设置、基本参数设置、时钟初始化
        存储器初始化
        硬件设备初始化
        跳转到第二阶段C程序入口点
      • 第二阶段（汇编、C）：
        处理器初始化、板级初始化、中断初始化等
        初始化本阶段要使用的硬件设备
        在RAM中运行操作内核：将内核代码、根文件系统映像从ROM复制到RAM
        向操作系统内核传递启动参数
        调用内核代码
    • Universal Bootloader：遵循GPL条款的通用bootloader
      支持PowerPC、ARM、x86系列处理器体系结构
      支持操作系统Linux、VxWorks、QNX
      支持启动加载、下载模式
    设备驱动程序：
    • 直接与硬件相互作用并控制硬件的软件，只能被高层软件调用，无法自行运行
    • 功能：设备初始化、设备打开/关闭、设备读取和写入、设备状态查询/控制操作
    • 实现方式：
      • 小型操作系统只包括操作系统内核，编写设备驱动程序无统一框架
      • 大型操作系统定义统一的设备驱动程序框架
      • Linux等类Unix操作系统将I/O设备抽象成设备文件
      • 驱动程序必须在内核态运行，用户应用程序必须通过驱动程序访问硬件
    • VxWorks的设备驱动程序：
      • 初始化部分
      • 函数功能部分：7个标准I/O函数的具体实现
      • 中断服务程序
    实时系统：
    • 必须在有限、确定时间内对外部事件作出相应的信息系统
    • 时限：相对时限与绝对时限
    • 属性：可靠性、可预测性、时间约束性、交互性
    • 任务类型：
      1. 硬实时任务：必须在给定时限内完成，超市会导致失败或致命错误
      2. 软实时任务：系统响应越快越好
      3. 准实时任务：通常允许偶尔错过最后期限
    • 实时操作系统（RTOS）：
      • 异步I/O和中断处理能力
      • 任务切换时间和中断延迟时间确定
      • 优先级中断和调度
      • 实时抢占式调度、内存锁定、连续文件、同步
      • 可对外部事件响应时间、实时任务执行时间进行判断
    • RTOS实时性指标：
      • 中断延迟时间（Interrupt Latency）
      • 任务切换时间（Task Switching Latency）
      • 响应时间：从事件请求到任务完成的时间间隔
    嵌入式操作系统：
    • 特点：代码固化存储、可裁剪性、实时性、强稳定性、弱交互性、硬件适应性
    • 分类：
      • 硬实时、软实时/非实时
      • 专门为嵌入式环境开发/从通用计算机操作系统移植
      • 免费/商业
        VxWorks按开发版本授权数量收取授权费，按产品售出数量收取版权费
    • 内核结构：
      • 单内核：把内核各功能模块整合，执行效率高
        Linux、Android、Mac OS
        • 占内存大，可扩展性低，可靠性低
        • 生成应用程序效率高，内核功能切换开销少
        • 操作系统内核运行效率高
      • 微内核：将必须的基本功能放入内核，其他功能在用户态运行的服务完成
        VxWorks
        • 接口一致，所有进程请求使用统一接口
        • 各功能模块间松散耦合，只完成服务功能
        • 功能扩展方便，各功能间切换开销大
    • 常用嵌入式操作系统：

      • Nucleus PLUS：抢占式多任务商业操作系统内核

      • TinyOS：伯克利开发，开源，为硬件资源有限的无线传感器网络应用定制
      • eCOS：低成本便携式设备（银行卡）

    • 特点：
      • 公开源码的高质量实时内核
      • 大部分代码用ANSI，C编写
      • 抢占式实时操作系统内核
      • 嵌套可达到255层
    • 基本功能：
      • 任务调度
      • 任务间通信
      • 同步
      • 任务管理
      • 时间管理
      • 内存管理
    • 任务构成：
      任务是操作系统的基本调度单位
      • 程序代码：无返回值的C函数
      • 任务堆栈：每个任务有自己独立的栈空间，各栈空间大小不一定相同
      • 任务控制块（TCB）：用于保存任务状态与属性的数据结构，任务创建时被初始化
    • 任务状态：
      • 休眠态：任务代码驻留在内存中，但未交给内核调度的状态
      • 就绪态：任务准备好，可运行，但被挂起的状态
      • 运行态：任务已经获得处理器使用权而正在运行的状态
      • 等待态：正在运行的任务因等待某事件发生而将处理器使用权让出的状态
      • 被中断态：处理器执行中断服务而被暂停的程序状态
    • 任务调度：
      系统可管理64个任务，用户任务最多56个
      • 空闲任务：没有其他任务处于就绪态时运行
      • 统计任务：每秒运行一次
      程序进入临界区时，不希望系统进行任务调度、处理器转入中断的解决方法：
      • 利用宏OS_ENTER_CRITICAL()与OS_EXIT_CRITICAL()实现开关中断
      • 利用函数OSSchedLock()与OSSchedUnock()给调度器上锁/解锁
    • 中断处理：
      OSIntNesting标识当前是否处于中断状态与中断嵌套层数（最高255）
      1. 保存CPU寄存器
      2. 调用OSIntEnter()或OSIntNesting直接加1
      3. 执行中断服务代码
      4. 调用OSIntExit()
      5. 恢复CPU寄存器
      6. 执行中断返回指令
    • 任务管理服务：
      • 任务创建：
        • 可在调用OSStart()前创建，不能由中断服务程序创建
        • OSTaskCreate()，OSTaskCreateExit()
      • 任务删除：
        • 使任务转入休眠状态，不被内核调度
        • OSTaskDel()：可删除任务自身与其他任务
      • 任务挂起与恢复：
        • OSTaskSuspend()：挂起任务
        • OSTaskResume()：恢复任务
        • 任务在被挂起时在等待延时，则挂起操作将被取消
      • 保护任务间共享数据和提供任务间通信的方法：
        1. 通过宏OS_ENTER_CRITICAL(), OS_EXIT_CRITICAL()开关中断
        2. 通过OSSchedLock(), OSSchedUnock对任务调度函数上锁/解锁
        3. 利用信号量、互斥信号量、邮箱、消息队列进行任务间通信
    • 时钟节拍与时间管理服务：
      • 在调用OSStart()启动多任务调度后开启时钟节拍器，初始化定时器中断
      • 处于运行态的任务可用OSTimeDly()，OSTimeDlyHMSM延迟进入等待状态，可用OSTimeDlyResume(INT8U prio)唤醒，prio为优先级
    • 任务间通信与同步服务：
      • 实现方法：信号量、消息邮箱与消息队列、互斥信号量、事件标志组
      • 任务/中断服务程序可通过事件控制块ECB向其他任务发出信号（中断服务程序不能等待信号）
      • 多个任务可同时等待同一事件的发生
      • 互斥信号量可解决优先级反转问题

    开发基础知识

    嵌入式系统开发过程：
    • 需求分析与规格说明
      需求包括功能部分、非功能部分
    • 系统设计
      确定实现功能的软件、硬件结构，与相应功能的划分
    • 构件设计
    • 系统集成与测试
      分部、按阶段进行测试（测试贯穿各阶段）
    嵌入式系统开发特点：
    • 使用宿主机-目标机的开发架构
    • 与底层硬件控制程序联系密切
    • 软硬件资源受限制
    • 需要固化程序
    开发平台与开发工具：
    • 作用：提高开发质量、缩短开发周期、降低开发成本
    • 典型开发环境：ARM，DSP，51
    嵌入式系统的调试：
    • 低端（8/16位）系统的调试可通过在线仿真器（ICE）实现
      被调试系统与ICE共用一个处理器
    • 片上调试技术JTAG
      • 在靠近芯片的I/O引脚处增加一个移位寄存器（边界扫描寄存器BSR）
      • 被调试的芯片还需有一个测试访问接口TAP，使宿主机上的调试程序可对扫描寄存器中内容进行I/O操作
      • JTAG标准为片上测试技术的一种国际标准，允许多个芯片的BSR通过JTAG串联
      • JTAG标准规定TAP控制器中有两种寄存器：数据寄存器（DR）、命令寄存器（IR）
    • JTAG适配器：
      • 大多数CPU，DSP，FPGA支持JTAG标准（51不支持）
      • 通过JTAG进行调试，无需目标存储器，不占用目标机任何I/O端口（串行）
      • JTAG支持多种芯片的烧写功能
    • 驻留监控软件调试：
      • 固化在目标机ROM中的程序，成本较低，不需要其他硬件调试和仿真设备（如ARM的Angel）
    • 指令集模拟器（ISS）：在宿主机上模拟应用程序在目标机上运行行为的软件工具（ARM的ARMulator，SkyEye）
      应用场合：
      • 无目标机开发板硬件或成本高
      • 程序模块不需要在实际开发板上执行
    ARM ADS：
    • 一个工程项目中至少应包含一个生成目标
      ARM提供的可执行输出文件模板包括
      • Debug生成的输出文件包含所有调试信息
      • Release生成的输出文件不包含调试信息，用于生成实际发行的软件
      • DebugRel生成的输出文件包含基本调试信息
    • 连接顺序：
      • 地址映射关系简单：使用编译、连接选项确定输入文件的连接顺序
      • 地址映射关系复杂：使用Scatter格式的文件（文本文件）确定连接顺序
        一个Scatter配置文件通常要描述1个下载时域、若干运行时域
    • 工程项目建立：
      添加已存在文件时：
      • 文件扩展名必须在文件映射表中有定义
      • 生成目标的输入文件在工程项目中不能重名
    • 连接中的地址映射：对代码或数据进行存储空间的地址分配
      编译后目标文件中的段（信息存储方式）
      • RO段：代码存储区和常数存储区，只读
      • RW段：初始值非0的全局变量数据区，可读可写
      • ZI段：初始值为0的全局变量数据区，包括未初始化的全局变量，可读可写
    • 下载时域：下载时的存储空间
      运行时域：运行时的存储空间
    • 编译与调试：
      • 编译新添加进入/修改的源文件，生成相应目标文件
      • 连接各目标文件与库文件，生成ELF格式的输出文件
      • 用fromELF转换输出文件的格式
    RVDS工具套件：基于ARM系列CPU开发的工具套件
    • 生成代码紧凑，执行效率高，支持51
    • 套件内容：
      • 工程管理器（IDE）
      • 编译连接器（RVCT）
      • 调试器（RVD）
      • 指令集仿真器（RVISS）
    • 功能：
      • Trace：记录ARM CPU从开始执行程序到停止所执行的所有动作，包括指令和主存储器访问
      • Profile：对Trace功能记录的内容进行性能分析
    GNU：
    • 常用于开发基于Linux操作系统的嵌入式软件的工具套件
    • 编译时在命令中加入_g参数，生成调试信息
    • GCC：GNU中的工具软件，针对Linux操作系统环境下应用程序的编译工具
      将源程序通过预处理、编译、汇编、连接，转化为可执行文件
      要生成可执行文件，所有被编译和连接的源文件中必须有且仅有一个main函数
    • GCC命令：
      基本格式：gcc [options] [filenames]
      针对不同硬件平台的目标机，基本命令格式不同（arm_linux_gcc）
      GCC命令中的参数：
      • -c 仅完成编译，不进行连接，生成.o目标文件
      • -E 仅完成预处理，不进行后续操作，生成.i文件
      • -S 仅完成预处理、编译，不进行后续操作，生成.S文件
      • -o filename_out 确定编译后输出文件名
      • -g 使编译器在编译时产生用于调试的符号表
      • -O 确定对程序进行优化编译
      • -I dirname 将目录名为dirname的目录加入程序头文件的目录列表
      • -fpic 产生位置无关的目标代码，用于构建共享函数库
    • GDB：程序调试工具，提供单步执行、断点执行功能
      GDB基本命令：
      • file 装入需要调试的可执行文件
      • run 执行当前被调试的文件
      • kill 终止当前被调试的程序运行
      • next 执行一行源代码，遇函数调用不进入函数内部
      • step 执行一行源代码，遇函数调用进入函数内部
      • break 在代码中设置断点，执行到断点时挂起
      • watch 观察变量的值
      • list 列出部分源代码
      • quit 终止GDB运行
    Web服务器：
    • 嵌入式Web服务器与通用Web服务器的区别：
      • 嵌入式服务器运行环境中资源较匮乏
      • 嵌入式服务器主要作用是控制、管理设备
      • 嵌入式服务器在系统中优先级较低，不能干扰主要任务的运行
    • 基于嵌入式Web服务的应用架构：
      • 提供一个基于远程控制端的操控方式
      • B/S操控方式：用户在远程的客户机上，通过浏览器与嵌入式Web服务器交互，操控被嵌入的设备
      • Web服务器通常与设备的控制系统合为一体
      • 支持以太网通信的电路：以太网控制电路、以太网驱动电路
