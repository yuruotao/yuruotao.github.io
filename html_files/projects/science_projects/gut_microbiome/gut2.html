    Abstract

    Statistical models are of vital importance in real-world applications.

    This article contains the statistical model framework with mathematical description and code realization accordingly.

    Data Preprocessing, Correlation Analysis, Clustering Analysis, Weight Determination, Tendency Estimation, Causality Determination, Network Analysis

    Statistics Terminologies

    Data Type:
    1. Quantitative:
      • Discrete
      • Continuous
    2. Qualitative:
      • Ordinal
      • Nominal
    Null Hypothesis: the claim that no relationship exists between two sets of data or variables being analyzed;
    Type 1 Error: the mistaken rejection of a null hypothesis as the result of a test procedure (False Positive);
    Type 2 Error: the mistaken failure to reject the null hypothesis as the result of a test procedure (False Negative);
    Covariance: a measure of the joint variability of two random variables, display the tendency in the linear relationship between the variables;
    Rank Correlation: any of several statistics that measure an ordinal association;
    Median Absolute Deviation: a measure of the variability of a univariate sample of quantitative data; M A D = m e d i a n ( x i − x ˜ )
    Weighted Correlation Network Analysis: a widely used data mining method especially for studying biological networks based on pairwise correlations between variables;
    Imputation: the process of replacing missing data with substituted values;
    Identifiability: a property which a model must satisfy for precise inference to be possible;
    Trimmed Estimator: an estimator derived from another estimator by excluding some of the extreme values;
    Bootstrapping: any test or metric that uses random sampling with replacement;
    Partial Correlation: measure the degree of association between 2 random variables with the effect of a set of controlling random variables removed;
    k-th Order Statistic: its kth-smallest value;
    Simpson's Paradox: a phenomenon that a trend appears in several groups of data but disappears or reverses when the groups are combined.

    Outline of Mathematical Model Establishment

    Open in viewer
    For a data-based mathematical model establishment, generally the purpose is to reconstruct the model as accurate as possible. The first step is to collect and preprocess the raw data since the statistical noise and other types of errors may invalidate the analysis afterwards. Then the correlation analysis is applied to the data. The reason to conduct correlation analysis on given data, assume we have an interested quantity refer as output, is to see the relationship among variables and the relationship between the output and variables. We refer the variables that are related to the output as indicators. When the correlation analysis is complete, we reduce the variables that are statistically irrelevant with the output. If the correlations among indicators are high, this indicates the indicators are interacting with each other. To see how much an indicator influences the output, we need to carry out weight calculation for the indicators. To see how indicators interact with each other, we need to carry out network analysis, as discussed in Network Analysis. For simplicity, we can also group the correlated indicators so that the new indicators can be viewed as irrelevant statistically. Grouping the indicators requires clustering analysis, and within each cluster, the weight should be calculated to synthesize an output. We will discuss the clustering methods in Clustering Analysis, weight calculation in Weight Determination.
    After applying the analysis approaches above, we are now able to analyze the status of a data-based system. For different systems, the way forward parted. For time-sensitive systems, the way forward is to estimate the developing tendency forward through time. For relationship-sensitive systems, the way forward is to determine causal relationships through correlation. Tendency Estimation discusses the approaches to make estimations on the trend of the model, and Causality Determination discusses the relationship between correlation analysis and causal analysis.
    And finally, we make assessments on the accuracy of the model and make corrections accordingly in Error and Correction. The notations to be used are summarized in Table 1, and a flowchart of the procedure is shown in Figure 1 above.

    Data Normalization

    In most cases, the data we obtain would not be in the same scale. By normalizing the data, we can establish a unified framework for all columns in our dataset. We introduce the method to normalize the data below.
    1. Z-score: center the data around mean with unit range of standard deviation z i = x i −μ σ
    2. Winsorization: set the extreme values larger than the selected bound to be the boundary value
    3. Center: shift by the length of mean
    4. Rank-transform: rank all the data and return the relative ranking 
                                                    # Data normalization with R
library("datawizard")

normalized_data_file1 = winsorize(data_file)
normalized_data_file2 = center(data_file)
normalized_data_file3 = ranktransform(data_file[, c(2)])
        # Data normalization with Python
def normalize(df):
    result = df.copy()
    for feature_name in df.columns:
        mean_value = df[feature_name].mean()
        sigma_value = df[feature_name].std()
        result[feature_name] = (df[feature_name] - mean_value)/sigma_value
    return result

def winsorize(df):
    from scipy.stats.mstats import winsorize
    result = df.copy()
    
    def using_mstats(s):
        return winsorize(s, limits=[0.01, 0.01])
    
    result = df.apply(using_mstats, axis=0)
    return result

def center(df):
    result = df.copy()
    for feature_name in df.columns:
        mean_value = df[feature_name].mean()
        result[feature_name] = (df[feature_name] - mean_value)
    return result

normalized_data_file = normalize(data_file)

    Correlation Analysis

If the change in one variable cause little or no fluctuation in the other one, it is likely that the two variables are not correlated.     P-value Adjustment with Package Correlation

    Here are the methods that used for adjusting p-values, they are applied to guarantee the significance of the result.

    Bonferroni Correction

    Parameter: "bonferroni"
    Definition: compensate the increase of rare events by testing each individual hypothesis at a significance level of α m .

    Holm Method

    Parameter: "holm"
    Definition: control the family-wise error rate. Assume having m p-values, P 1 ,..., P m ,(ranked in an increasing order) corresponding to hypothesis H 1 ,..., H m , the FWER should be lower than α .
    For each p-value, test whether P i < α m+1−k If so, reject the corresponding hypothesis.

    Hochberg's Step-up Procedure

    Parameter: "hochberg"
    Definition: more powerful than Holm's procedure, can be regarded as a step-down version of the Bonferroni test.
    Assume having m p-values, P 1 ,..., P m ,(ranked in an increasing order) corresponding to hypothesis H 1 ,..., H m . Let R be the largest k such that P (k) < α m+1−k Reject the null hypotheses H (1) ,..., H (R)

    Benjamini-Hochberg Procedure

    Parameter: "BH"
    Definition:
    1. For a given α , find the largest k such that P (k) ≤ αk m
    2. Reject the null hypotheses for all H (1) ,..., H (k)

    Parameter: "hommel"
    Definition: more powerful than Hochberg.

    2. For all i , if no such k exists, reject all hypotheses. Otherwise, reject all H i with P i ≤ α k

    Clustering Analysis

    In clustering analysis, the first step is to define a measure of distance, which is used to describe the closeness among indicators. Then the goal of which value to optimize should be established, whether it is the distance among clusters or the distance of all indicators to the center of their cluster. There is no fit-all clustering method, only different approaches for different goals. We introduce the measures of distance first, then the goals to optimize before diving into the clustering methods.

    Measures of Distance

    The measures of distance are similar to correlation coefficients, portraying the similarity of data. We list the commonly applied distances below.
    1. Manhattan Distance
    2. Euclidean Distance
    3. Chebyshev Distance
    4. Minkowski Distance
    5. Camberra Distance

    Manhattan Distance

    Parameter: dist(data, method="manhattan")
    Definition: d( x → , y → )= x → − y → = ∑ i=1 n x i → − y i →

    Euclidean Distance

    Parameter: dist(data, method="euclidean")
    Definition: d( x → , y → )= ∑ i=1 n ( x i − y i ) 2

    Chebyshev Distance

    Usage: the distance between two vectors is the greatest of their differences along any coordinate dimension
    Parameter: dist(data, method="maximum")
    Definition: D C ( x → , y → )= max i ( x i − y i )

    Minkowski Distance

    Usage: a generalization of both the Euclidean distance and the Manhattan distance
    Parameter: dist(data, method="minkowski")
    Definition: D M ( x → , y → )= ( ∑ i=1 n x i − y i p ) 1 p

    Camberra Distance

    Usage: a weighted version of Manhattan distance
    Parameter: dist(data, method="canberra")
    Definition: d( x → , y → )= ∑ i=1 n x → − y → x → + y →

    Clustering Methods

    When considering the relationships in clustering analysis, 2 types of relationships emerge. One is the relationship within the clusters, the other one is the relationship among the clusters. We can describe such relationships with the words compactness and connectedness. Methods that focus on compactness seek to minimize the distances within clusters, and methods focus on connectedness seek to maximize the dissimilarity among clusters. Thus, the methods for clustering analysis are categorized into 2 types:
    1. Compactness-based:
